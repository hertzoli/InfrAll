# Test Scope

## Prioridade alta

- validacoes;
- regras de negocio;
- transformacoes de dados;
- servicos independentes;
- casos de erro e borda.

## Prioridade baixa

- evento visual puro;
- layout de formulario;
- integracao direta com Designer;
- detalhes cosmeticos da UI.

## Regra pratica

Quanto mais a logica puder viver fora do formulario, maior o retorno do teste automatizado.
