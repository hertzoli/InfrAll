# Nested Task List

Criar lista hierarquica em markdown com foco em execucao real.

## Regras

- Quebrar por entrega funcional, nao por intuicao vaga.
- Manter tarefas pequenas o suficiente para serem verificadas.
- Usar subtarefas quando houver dependencia tecnica clara.
- Ordenar em sequencia que preserve build e comportamento.

## Estrutura recomendada

- tarefa principal;
  - subtarefa de analise;
  - subtarefa de implementacao;
  - subtarefa de integracao;
  - subtarefa de validacao tecnica.

## Maus sinais

- tarefa enorme sem fronteira clara;
- subtarefa que nao leva a uma entrega verificavel;
- plano que nao conversa com o codigo real.
